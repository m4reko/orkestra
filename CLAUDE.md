# CLAUDE.md

## Project

Internal member and project management system for Uppsala Blåsarsymfoniker (wind orchestra). See `docs/mvp-spec.md` for features and `docs/technical-spec-gleam.md` for architecture.

## Stack

- **Gleam** (on BEAM/Erlang) with Wisp, Mist, Lustre SSR, hx, HTMX 4, Tailwind CSS
- **SQLite** via sqlight (raw SQL, no ORM)
- **sqlgen** code generator: reads `sql/*.sql` → `src/orchestra/generated/sql.gleam`

## Tooling

| Tool | Command | Purpose |
|------|---------|---------|
| gleam | `gleam build` | Build project |
| gleam | `gleam run` | Run dev server (port 8000) |
| gleam | `gleam test` | Run tests |
| gleam | `gleam add <pkg>` | Add dependency, prefer this over modifying gleam.toml |
| gleam | `gleam format` | Format code |
| sqlgen | `gleam run -m sqlgen` | Regenerate SQL query module from `sql/` |
| Tailwind | `tailwindcss -i priv/static/css/input.css -o priv/static/css/output.css` | CSS compilation |

## Getting started

```sh
gleam run                                        # Start server (downloads deps, compiles, runs migrations)
sqlite3 orchestra.db < scripts/seed_dev_data.sql # Seed dev data (5 test members)
```

The server creates `orchestra.db` and runs migrations automatically on startup. Seed data is optional — run the seed script to populate test members.

## Project structure

Follows [jsorm](https://github.com/aosasona/jsorm) conventions:

- `src/orchestra/web/` — HTTP request handlers (one per domain)
- `src/orchestra/pages/` — Lustre HTML builders (full pages + HTMX fragments)
- `src/orchestra/models/` — Data types, decoders, and DB operations
- `src/orchestra/lib/` — Shared utilities (response helpers, auth, validation)
- `src/orchestra/generated/` — Auto-generated by sqlgen (do not edit)
- `sql/` — Raw SQL query files (one per query)
- `priv/migrations/` — Schema migration SQL files
- `priv/static/` — htmx.min.js, compiled CSS

## Code style

- **Functional style** — natural in Gleam, everything is functions.
- **Swedish UI, English code.** User-facing text in Swedish. Variable names, comments, and git messages in English.
- **Raw SQL.** Queries in `sql/` files, generated into typed Gleam functions via sqlgen. Dynamic queries (optional filters) built in model functions.
- **HTMX 4.** Routes return HTML fragments for HTMX requests, full pages otherwise.
- **Result types for errors.** No panics in application code. `let assert` only for startup.
